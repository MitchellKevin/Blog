Vakken & Projecten / HCD
HCD — VoiceReader
// vak 03 · hcd
Human Centered Design
VoiceReader is een Chrome-extensie gebouwd rond de behoeften van één echte gebruiker: Ihab is blind en gebruikt de hele dag een screenreader. Zijn grootste frustratie was niet de snelheid, maar de vlakke, emotieloze stem. VoiceReader detecteert met AI het genre van een pagina en leest die voor in een bijpassende, emotionele stem.
Bekijk live site
VakHuman Centered Design
ProjectVoiceReader
TypeChrome-extensie
Getest metIhab (blind) · 4 weken
AIElevenLabs · Gemini
De gebruiker centraal
Het hele ontwerp is samen met Ihab tot stand gekomen — een blinde hoofdgebruiker. Vier weken lang testen op mijn eigen laptop, met stemstijlen afgestemd op zijn interesses. Niet ontwerpen vóór, maar mét.
Hoe het werkt
AI bepaalt het genre van een pagina; ElevenLabs TTS leest die voor met één stem per genre en stijl. Zo krijgt een verhaal een andere voordracht dan een nieuwsartikel.
De techniek
De eerste test met Ihab leidde tot genre-detectie via Claude. Toen Claudes credits op waren ben ik overgestapt naar Google Gemini en heb ik alles teruggebracht naar de extensie zelf.
Wat ik leerde
HCD is ontwerpen mét je gebruiker, niet voor een aanname. Verder technisch: Gemini's REST-API gebruikt camelCase en de modelnaam verschilt per API-versie, dus auto-discovery van het model scheelt gedoe. En een echte accessibility-first interface bouwen is iets anders dan "toegankelijk genoeg".

API — CMD Casino
// vak 04 · api
API
Voor het API-vak bouwde ik CMD Casino: een full-stack multiplayer casino. Spelers maken een account aan, verdienen credits en spelen real-time Blackjack tegen elkaar. API-gedreven in Astro met één content-API en twee web-API's, gedeployed op Render.
Bekijk live site
VakAPI
ProjectCMD Casino
StackAstro · Node.js · MongoDB
RealtimeWebSockets
BetalingenMollie (iDEAL)
HostingRender
De API's
Een content-API serveert de in-game spelregelpagina's, gevoed door de Wikipedia-API. De Mollie-API regelt iDEAL-betalingen: Mollie bevestigt een betaling door een webhook op mijn server aan te roepen.
Real-time multiplayer
Via WebSockets spelen meerdere mensen aan dezelfde Blackjack-tafel, live. Een pratende dealer maakt de tafel af.
Server-authoritative & anti-cheat
Alle spellogica draait op de server, niet in de browser. Daardoor kan een speler de kaarten niet manipuleren — de client vraagt alleen, de server beslist.
Wat ik leerde
Wikipedia-artikelen zijn enorm en vol ruis, dus ik cache ze en roep de API nooit twee keer aan voor hetzelfde artikel. Verder: webhooks afhandelen, consistent camelCase aanhouden zoals de API doet, en deployen op Render. API-kosten zijn een echte drempel — caching is dan niet luxe maar noodzaak.

Vakken & Projecten / Sprint 0
Sprint 0
// project · feb 2026
Sprint 0
De introductie-opdracht: maak in 9 dagen een one-page website over jezelf, met je leerdoelen erin. Ik wilde iets nieuws leren en koos voor 3D — Three.js en WebGL. Wat begon als een 3D-portfolio werd uiteindelijk een interactief Monopoly-speelbord over mijn leven.
Bekijk live site
OpdrachtWebsite over jezelf
Periode3–11 feb 2026
TechniekenThree.js · WebGL
Het 3D-portfolio
Three.js leren door te doen: .glb-modellen van Sketchfab en Poly, en leren werken met camera, lichten, canvas en het comprimeren van modellen. Daarna interactie op scroll — een tennisracket dat een slag maakt en een auto die het scherm in rijdt met draaiende wielen. Veel wiskunde, want het draaipunt van het racket zat in het midden in plaats van aan het uiteinde.
De pivot naar Monopoly
Een nieuw concept: een speelbord waarbij de straten thema's uit mijn leven zijn — sport, studie, hobby, games, muziek. Met 3D-dobbelstenen, een lopende pion en speelkaarten die hun inhoud uit een eigen API halen.
Wat ik leerde
De GLTFLoader gaf de hele dag 404's, dus ben ik teruggevallen op simpele Three.js-vormen — niet blijven hangen, doorbouwen. Het eureka-moment: niet het bord draaien bij een bocht, maar de camera. En ik bouwde een eigen API van ~260 regels toen die van de opleiding nergens te vinden was. Leerdoelen: ontwerpen met 3D, betere animaties met wow-factor, en interactie waarbij de gebruiker voelt dat híj de site bestuurt.
